上一篇博客讲的是针对具有一些固定映射的自定义字体，即能够使用该固定映射来找到当前网页所使用的自定义字体的映射表，从而替换使用。然而，在一些网站中，其根本找不到什么规律，每次都是随机的字符映射表，本篇博客将以猫眼为例，讲述遇到这种随机的字符映射表，我们应该处理。

首先，我们根据之前说讲的方法，找到当前网页所使用的自定义字体文件的地址，并将其转化为xml文件进行查看，如下：

同样发现找不到任何规律。那对于这样的网页我们该如何处理呢？在上一篇中我们有提到一个字体其具有一定的坐标，并通过这个坐标来将字给显示出来，所以，我们可以对于这种随机的字符映射表，我们也可以根据坐标来找到其对应的字符映射表。

我们将两个字体文件中的0都拿出来看一下（在xml文件中观察），其坐标信息是否完全相同

字体1中0：

字体2中0：

发现俩个字体的坐标对象是相同的，因此，到这里我们变可以知道，通过一个基本的字体文件中的字体的坐标对象与当前网页中得到的字体文件中的字体的坐标对象进行比较便可以得到当前字符映射表。当然，我们对于这个基本的字体文件需要进行相应的处理。我们需要将其name对应的字符映射给写出来，以供我们对当前网页中的字体映射进行替换使用，得到的映射表信息为（这个需根据自己第一次下载下来的字体文件进行修改）

对于当前网页中自定义字体的处理步骤如下：

  • 将当前网页中的自定义字体文件地址找到并下载保存
  • 得到当前自定义字体文件中所有的code:name
  • 根据上一步中得到的映射表遍历(根据name)当前字体文件glyf中的字体坐标对象，并与之前所得到的用来比较的字体文件中的name所对应的字体对象进行比较，当两个对象相等时，变可以通过之前得到的基本字体文件映射表得到当前字体文件的映射表
  • 根据网页源码中的显示将字符映射表的格式进行相应的转化
  • 将网页中的code根据字符映射表替换为真实字符
